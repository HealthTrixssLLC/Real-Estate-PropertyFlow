We need to convert the `artifacts/tour-mobile` app from an Expo-managed React Native app into a native Xcode-buildable iOS app without Expo.

Context:
The repo is `Real-Estate-PropertyFlow`.
The mobile app is located at `artifacts/tour-mobile`.
The root workspace currently builds successfully with:

```bash
pnpm install
pnpm run build
```

However, when attempting to build the iOS app locally with Xcode/Expo, the build fails inside the Expo native dependency `expo-file-system`:

```text
expo-file-system/ios/FileSystemModule.swift
value of type 'any EXFileSystemInterface' has no member 'getPathPermissions'
CommandError: Failed to build iOS project. "xcodebuild" exited with error code 65.
```

Goal:
Remove Expo from the mobile app and make `artifacts/tour-mobile` a bare React Native iOS app that can be opened and built directly in Xcode using an `.xcworkspace`.

Important:
Do not break the rest of the monorepo.
Do not change unrelated packages.
Do not require Expo CLI, Expo Go, EAS, or `expo prebuild` for local iOS development.
The final iOS app should build using Xcode and CocoaPods.
